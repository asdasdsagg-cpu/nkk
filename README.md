# Топливо рядом

Единое Next.js-приложение: лендинг, интерактивная карта АЗС и серверные API-маршруты для получения данных «ГдеБЕНЗ».

## Локальный запуск

Требуется Node.js 22.

```bash
npm ci
npm run dev
```

Откройте `http://localhost:3000`. Карта доступна по адресу `http://localhost:3000/map`.

## Проверка production-сборки

```bash
npm run build
npm start
```

## Развёртывание на Render

1. Загрузите проект в GitHub/GitLab/Bitbucket.
2. В Render выберите **New → Blueprint** и подключите репозиторий.
3. Render прочитает `render.yaml`, соберёт приложение и запустит один Node.js Web Service.

### Уведомления безопасности в Telegram

В настройках сервиса Render задайте две секретные переменные окружения:

- `TELEGRAM_BOT_TOKEN` — новый токен, полученный у `@BotFather`;
- `TELEGRAM_CHAT_ID` — ID чата, куда бот уже добавлен и может отправлять сообщения.

После сохранения переменных выполните новый deploy. Сайт отправляет не более одного уведомления с одного IP за 10 минут и не более одного раза за вкладку/сессию браузера. Токен используется только серверным API и не передаётся посетителю.

Если токен когда-либо публиковался в сообщении, журнале или репозитории, сначала отзовите его у `@BotFather` командой `/revoke` и используйте только новый токен.

Дополнительные сервисы и секреты не требуются. Проверка работоспособности доступна по `/api/health`.

## Структура

- `app/` — страницы и API;
- `public/` — изображения и локальные снимки данных;
- `render.yaml` — конфигурация Render;
- `package.json` — команды запуска и зависимости.

Данные внешнего источника могут временно быть недоступны; интерфейс корректно показывает локальный снимок или сообщение об ошибке.
